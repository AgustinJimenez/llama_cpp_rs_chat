Dialog Test

This is a simple test to verify the dialog API changes work correctly.

Changes Made:

  • Installed @tauri-apps/plugin-dialog package
  • Added dialog plugin to Cargo.toml and main.rs
  • Updated ModelConfigModal.tsx to use Tauri's dialog API instead of HTML file input
  • Fixed file path handling to store full absolute paths instead of just filenames

Key Fix:

Previously: setModelPath(file.name) - Only stored filename

Now: setModelPath(filePath) - Stores full absolute path

Expected Behavior:

When users click "Browse" button, it will:

  1. Open Tauri's native file dialog
  2. Filter to show only .gguf files
  3. Return the full absolute path when file is selected
  4. Store the complete path (e.g., "E:\repo\llama_cpp_rs_chat\assets\test-models\test.gguf") instead of just "test.gguf"

This should fix the "does not exist" error when loading models.
